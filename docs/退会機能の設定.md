# 退会機能の設定

## 仕様

- ユーザーが「退会する」を実行すると、**プロフィール・コスメリスト・認証アカウント**が削除され、同じメール・Google・X では再ログインできなくなります。

## 必要な設定

退会時に**認証アカウントまで削除**するには、次の環境変数を設定してください。

| 環境変数 | どこで取得するか |
|----------|------------------|
| `SUPABASE_SERVICE_ROLE_KEY` | Supabase ダッシュボード → **Settings** → **API** → **Project API keys** の **service_role**（secret）をコピー |

- **anon key**（公開用）とは別の **service_role** の鍵を使います。
- `.env.local` に `SUPABASE_SERVICE_ROLE_KEY=貼り付けた鍵` と追加してください。
- この鍵は**サーバー側だけで**使い、クライアントのコードや Git にコミットしないでください。

## 設定していない場合

`SUPABASE_SERVICE_ROLE_KEY` を設定していない場合でも、退会処理は動きますが、**DB のデータ（プロフィール・コスメなど）だけが削除**され、**認証ユーザーは残ります**。そのため、同じメール・Google・X で再度ログインすると、空の状態でアカウントを再利用できます。  
認証まで消したい場合は、上記のとおり `SUPABASE_SERVICE_ROLE_KEY` を設定してください。
