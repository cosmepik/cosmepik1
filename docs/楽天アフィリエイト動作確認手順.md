# 楽天アフィリエイト・レベニューシェア 動作確認手順

楽天アフィリエイトと確率分散型レベニューシェアが正常に動作しているか確認する方法です。

---

## 1. 環境変数の確認

以下が `.env.local` または Netlify の環境変数に設定されているか確認してください。

| 変数名 | 用途 | 必須 |
|--------|------|------|
| `NEXT_PUBLIC_ADMIN_RAKUTEN_ID` | 運営の楽天アフィリエイトID（例: `0ea12345.ab.cd`） | レベニューシェア用 |
| `NEXT_PUBLIC_USER_REVENUE_SHARE_RATE` | ユーザーへの分配率（0.6 = 60%） | 任意（未設定時は0.6） |
| `RAKUTEN_APPLICATION_ID` | 楽天API検索用 | 商品検索を使う場合 |
| `RAKUTEN_ACCESS_KEY` | 楽天API検索用 | 商品検索を使う場合 |

**デバッグAPIで確認:**

```
GET /api/affiliate/debug?check=env
```

レスポンスで `adminIdSet` が `true` か確認します。

---

## 2. アフィリエイトリンク生成の確認（デバッグAPI）

確率分散のロジックが正しく動いているか、デバッグAPIで確認できます。

```
GET /api/affiliate/debug?itemUrl=https://item.rakuten.co.jp/xxx/yyy/&userAffiliateId=0ea99999.ab.cd&slug=demo
```

**パラメータ:**
- `itemUrl` … 楽天商品URL（必須）
- `userAffiliateId` … ユーザーの楽天アフィリエイトID（任意。未設定＝運営100%）
- `slug` … プロフィールのslug（任意）

**レスポンス例:**
```json
{
  "ok": true,
  "generatedUrl": "https://hb.afl.rakuten.co.jp/hgc/0ea12345.ab.cd/?pc=...",
  "usedId": "admin",
  "userAffiliateId": "0ea99999.ab.cd",
  "adminIdSet": true,
  "userRate": 0.6
}
```

- `usedId: "user"` … ユーザーIDでリンク生成（ユーザー60% / 運営40%）
- `usedId: "admin"` … 運営IDでリンク生成（運営100%）

何度かリクエストして、`userAffiliateId` を渡した場合に `usedId` が `user` と `admin` の両方出ることを確認すると、確率分散が動いている証拠になります。

---

## 3. クリック時の動作確認

1. 公開ページ（例: `/p/あなたのslug`）を開く
2. 商品カードまたはリンクをクリック
3. 新しいタブで楽天の商品ページが開く
4. アドレスバーのURLを確認
   - `https://hb.afl.rakuten.co.jp/hgc/あなたのID/` または `.../運営ID/` で始まっていればOK

**ユーザーID未設定時:** 常に運営IDのリンクになる  
**ユーザーID設定時:** 60%でユーザーID、40%で運営IDのリンクになる（確率）

---

## 4. クリックログの確認（Supabase）

クリックが記録されているか、Supabase ダッシュボードで確認できます。

1. Supabase ダッシュボード → Table Editor
2. `click_logs` テーブルを開く
3. 商品リンクをクリックしたあと、新しい行が増えているか確認

**カラム:**
- `username` … プロフィールのslug
- `item_id` … 商品ID
- `product_url` … 元の商品URL
- `used_id` … `user` または `admin`（どちらのIDでリンクを生成したか）
- `created_at` … クリック日時

---

## 5. 楽天アフィリエイト管理画面での確認

実際の成果（クリック数・売上）は、楽天アフィリエイトの管理画面で確認します。

1. [楽天アフィリエイト](https://affiliate.rakuten.co.jp/) にログイン
2. レポート → アクセス解析 などでクリック数・成果を確認
3. 運営IDとユーザーIDそれぞれで、クリックが計上されているか確認

※ 成果報酬は購入が発生してから反映されるため、即時には表示されません。

---

## 6. よくある問題

| 症状 | 原因 | 対処 |
|------|------|------|
| リンクが `hb.afl.rakuten.co.jp` にならない | `NEXT_PUBLIC_ADMIN_RAKUTEN_ID` 未設定または形式不正 | 正しいID（例: `0ea12345.ab.cd`）を設定 |
| 常に運営IDになる | ユーザーの `rakuten_affiliate_id` が未設定 | 収益化ページで楽天アフィリエイトIDを入力 |
| クリックログが増えない | Supabase 未接続 / `click_logs` テーブルなし | マイグレーション実行、接続確認 |
| 楽天以外のURLをクリック | 楽天商品URL以外はそのまま開く仕様 | 正常動作（Amazon等はアフィリエイト変換しない） |
