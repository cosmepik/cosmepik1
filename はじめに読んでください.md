# Cosmetree の動かし方（初心者向け）

このプロジェクトを自分のパソコンで動かすための手順です。  
「ターミナル」や「コマンド」が初めてでも大丈夫なように、順番に説明します。

---

## 事前に必要なもの

このプロジェクトを動かすには **Node.js** というソフトが必要です。

### Node.js が入っているか確認する

1. **ターミナル（黒い画面）を開く**
   - Mac: 「Spotlight」（画面右上の虫眼鏡）で「ターミナル」と入力して Enter
   - または「アプリケーション」→「ユーティリティ」→「ターミナル」

2. **次の文字をそのまま入力して Enter を押す**
   ```bash
   node -v
   ```
   - 何か数字（例: `v20.10.0`）が表示されれば **入っています** → 次の「動かし方」へ進んでOKです。
   - 「command not found」などと出たら **入っていません** → 下の「Node.js を入れる」をやってください。

### Node.js を入れる（入っていない人だけ）

1. ブラウザで https://nodejs.org/ を開く
2. **LTS** と書いてある方の「ダウンロード」をクリック
3. ダウンロードしたファイルを開いて、画面の指示に従ってインストール
4. インストールが終わったら、**ターミナルを一度閉じて、もう一度開き直す**
5. もう一度 `node -v` を実行して、数字が表示されればOKです

---

## 動かし方（3ステップ）

### ステップ1: プロジェクトのフォルダに移動する

ターミナルで、このプロジェクトがある場所に移動します。

1. ターミナルを開く
2. 次のコマンドを **1行ずつ** 入力して Enter（あなたのパソコンの場所に合わせてください）
   ```bash
   cd /Users/jojiyamaguchi/Downloads/cosmetree4
   ```
   - `cd` は「このフォルダに移動する」という意味です。
   - プロジェクトを別の場所に置いた場合は、その場所のパスに書き換えてください。

「何も表示されない」で次の行に進んでいれば成功です。

---

### ステップ2: 必要なパッケージをインストールする

初めて動かすときは、必ず1回だけこの作業をします。

1. 次のコマンドを入力して Enter
   ```bash
   npm install
   ```
2. 数十秒〜1分ほど待つ（たくさん文字が流れますが、そのままで大丈夫です）
3. 流れが止まって、また入力できる状態になればOKです

---

### ステップ3: 開発サーバーを起動する

1. 次のコマンドを入力して Enter
   ```bash
   npm run dev
   ```
2. 次のような表示が出れば成功です
   ```
   ▲ Next.js 15.x.x
   - Local:        http://localhost:3000
   ```
3. **このターミナルは閉じないでください。** 閉じるとアプリが止まります。

---

## ブラウザで開く

1. ブラウザ（Chrome、Safari など）を開く
2. アドレス欄に次のアドレスを入力して Enter
   ```
   http://localhost:3000
   ```
3. Cosmetree のトップページが表示されれば完了です。

- **トップ**から「インフルエンサー：検索・追加」をクリックすると、コスメを検索してリストに追加できます。
- 「管理画面」で並び替えや削除、「ファン向け：公開ページ（デモ）」で公開用の見た目を確認できます。

---

## 止め方

- アプリを止めたいときは、ターミナルで **Ctrl + C**（Mac では Control キーを押しながら C）を押します。
- もう一度動かしたいときは、もう一度 `npm run dev` を実行すればOKです。

---

## よくあること

| 状況 | 対処 |
|------|------|
| `npm install` でエラーが出る | 先に `node -v` で Node.js が入っているか確認する。入っていなければ Node.js をインストールする。 |
| `localhost:3000` でページが開かない | ターミナルで `npm run dev` が動いたままか確認する。止まっていたらもう一度 `npm run dev` を実行する。 |
| ターミナルで「パスが違う」ようなエラー | `cd` のあとのパスが、実際のプロジェクトの場所と一致しているか確認する。 |

不明な点があれば、どのステップで何が起きたかをメモして、誰かに聞くと伝わりやすいです。
